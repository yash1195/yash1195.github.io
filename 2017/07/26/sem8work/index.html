Classification of User Behaviour for Software Applications using Markov Chains

Yash Choukse
2017-07-26

Introduction

Any process satisfies the Markovian property if one can make predictions about the future of the process given only the present state.

While the underlying nature of many applications is rarely markovian but it is nevertheless a good assumption to model them as Markovian processes. When a user is using an application, it can safely be assumed that the interaction of the user with the application is Markovian in nature. In this project, we focus on the problem of classifying the behaviour of an unknown user based on the usage trails generated by the user while using any software application.

We study how the underlying Markov chains in an applicationâ€™s user base can be used to model different kinds of users. These user models can then be used to make a good prediction of which class the user of the application belongs to given only the current usage trails.

In this project, we construct Markov chains that model the different kinds of users of the application. With the proposed methodology, predictions can be made about the class of the user using the software application. As the usage trail increases, the prediction probabilities become more accurate.

The white paper can be found here

MarkPy

To use the proposed methodology in an application, a python library called MarkPy has been written. MarkPy provides a complete API for processing of software application data logs to process transitions of the known users of the software application.

Available on GitHub: MarkPy

The work is primitive but I hope developers contribute to it.
